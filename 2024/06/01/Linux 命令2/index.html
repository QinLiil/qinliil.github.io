文件查询

linux 分页查询

more -c [文件名]

more -c -3 [文件名]

  • 进行分页显示 刷新之前内容

file [文件所在路径]

文件摘选

head -n3 [文件名]

  • 查看文件前三行

tail -n3 [文件名]

  • 查看文件后三行

find查找指定目录下文件

使用语法：find [文件路径] [需查找文件名].

  • -name 按照文件名查找文件
  • -perm 按照文件权限来查找文件
  • -user 按照文件属主来查找文件
  • -mtine -n +n 按照文件的更改时间来查找文件

which

查找命令是否存在，以及执行的到底是哪一个位置的命令

文件压缩与打包

gzip 用来解压与压缩文件

使用语法：gzip [可选参数] [文件或目录]

  • -d 解开压缩文件
  • -f 强行压缩文件
  • -l 列出压缩文件的相关信息
  • -r 递归处理，将目标目录下的所有文件及之目录一并处理
  • -v 显示命令执行过程
    • 缺点无法压缩目录

tar 可打包多个文件打包到一个文件内

使用语法：tar [可选命令] [压缩包名] [文件名]

  • -c 建立新的压缩文件
  • -x 从压缩的文件中提取文件
  • -t 显示压缩文件内容
  • -z 选择gzip解压文件
  • -j 选择bzip2解压文件
  • -v 显示过程
  • f 备份一次文件

unzip 解压zip文件

使用语法：unzip [参数] 压缩包名

  • -d 指定解压路径
  • -o 解压不覆盖已有文件
  • -v 查看详细
  • -t 测试压缩文件是否有损坏
  • -x 指定文件 解压文件但不解压指定文件

文件命令·连接文件

ln 命令用于创建链接文件，默认情况下创建硬链接

使用语法： ln [-f |-n] [-s] 源文件 目标文件

  • -b 删除，覆盖以前建立的链接
  • -d 允许超级用户制作目录的硬链接
  • -f 强制执行
  • -l 交互模式，文件存在则提示用户是否覆盖
  • -n 把符号链接视为一般目录
  • -s 软连接（符号链接）必须绝对路径

连接文件

软连接 硬链接
以路径形式存在 以文件副本形式存在，但不占用实践空间
删除文件后链接失效 删除源文件无影响
可对目录进行连接 不可以对目录进行连接
可以跨文件系统 不可以跨文件系统

vim 基础操作

  • gg 到文件头部
  • G 到文件尾部
  • set nu 显示文件行数
  • y[1]w 可复制一个词（指两个空格间的文本）
  • :/word 在光标之后查找一个字符串word ,按n重复操作，shift + n 反向查找
  • :?word 在光标之前查找一个字符串word
  • :s/word1/word2/ 在光标之前替换第一个word1
  • :s?word1?word2 在光标之后替换第一个word1
  • :1,5s/word1/word2/g 将文件1-5行的word1替换为word2，不加g则只能替换每行的第一个
  • %s/word1/word2/gi 将文件所有的word1替换为word2，不区分大小写

提取列或字段

使用语法：cut [参数] [文件]

  • -b 显示行中指定直接范围的内容
  • -c [范围] ：仅显示行中指定范围的字符
  • -d：指定字段的分隔符，默认的字段分隔符为”TAB”
  • -f [范围]：显示指定第num个字段的内容，可以用逗号隔开显示多个字段

rpm 软件安装与管理

使用语法：rpm[参数] [安装包]

  • -i 指定安装的软件包
  • -h 所有“#”显示rpm详细的安装过程及进度
  • -v 显示安装的详细过程
  • -u 升级指定的软件包（卸载重装）
  • -f 升级指定的软件包（在原有的基础上安装）
  • -a 查看系统已安装的所有软件包
  • -V 查询已安装的软件包的版本信息
  • -p 查询/效验一个软件包的文件
  • -e 卸载软件包(nodeps)强制卸载软件可能会导致其他软件无法正常使用

rpm软件查询

使用语法：rpm [参数] [文件包]

  • -q 查询软件包是否安装
  • -qa 查询所有安装的软件包
  • -qf 查询所有已安装过的软件包
  • -qp 查询未安装的软件包
  • -ql 查询已安装的软件包的文件列表和完整的目录
  • -qi 查询软件包的详细信息
  • -qc 查询已安装软件包中的配置文件
  • -qd 查询已安装软件包中的帮助文档

dnf软件安装与管理

所有语法：dnf [参数] [文件包]

  • cachedir 缓存目录
  • best 升级安装包时默认安装最新版本，如无法安装则提示原因
  • info 查看rpm包的详细信息
  • list all/[软件包名] 查看软件包
  • downloab 下载软件包
  • install 安装软件包
  • remobe 卸载软件包

dnf check-update

dnf update package_name /dnf group update group_name

dnf软件原的启用与禁止

磁盘管理

挂载磁盘fastab

使用语法：fstad [参数] [目录]

  • auto 在启动时自动使用mount -a 命令挂载
  • ro 只读形式挂载
  • rw 读写模式挂载文件
  • user 允许任意用户挂载改文件
  • nouser 只能被root 挂载
  • dev/nodev 解析/不解析文件系统上的特殊设备
  • noatime/nodiratime 不更新文件系统/目录上inode 访问记录，可以提升性能
  • defaults 使用文件系统默认挂载参数

dump

  • dump 0/1 0表示忽略，1则经行备份。大部分用户没有安装dnmp，对他们而言应该设置为0
  • pass 1,2,3 根目录应当获取得最高得优先权 1，其他所有需要被检查得设备为2，0表示设备不会被fsck检查

分区工具使用

使用语法：fdisk [参数] [设备]

  • -b 指定分区大小
  • -l 列出指定得外围设备得分区表
  • -s 分区编号 将指定得分区输出到标准输出上
  • -u 搭配-l参数列表，会用分区数目取代柱面数目，来表示每个分区的起始地址
  • -v 显示版本信息

fdisk 分区工具使用流程

n:新建一个分区>p：选择主分区>输入1-4建立分区号>回车建立默认起始柱面> +大小 ：定义分区大小> p：打印分区信息>w：保存

fdisk修改分区类型

进入改硬盘 fdisk [硬盘位置] > t > 需要修改分区的id > L 查看类型有哪些

gpt 分区

删除分区：parted 硬盘所在位置 rm 硬盘ID

交互式硬盘分区

  • mklable gpt 创建gpt分区表，2t以上只能使用gpt,其他
  • mkpart 新建分区，不带指令parted会一步步提示用户输入相关信息
  • q 完成后保存退出

非交互式

  • parted [硬盘位置] [硬盘格式] :硬盘格式变成对应格式
  • parted [硬盘位置] mkpart [分区名] [起始-终止] :分区
  • mkfs -t ext4 [硬盘位置] :格式化

查看硬盘格式

parted -l

逻辑卷的管理

管理逻辑卷-创建物理卷

创建pv需要硬盘类型为8e

使用语法：pvcreate [参数] [设备路径]{设备id}

  • -f 强制创建物理卷
  • -u 指设备的uuid
  • -y 所有的问题都回答yes
    • pvs 显示当前物理卷的信息

创建LVM卷组(VG)

使用语法：vgcreate [参数] [卷组名] [物理卷列表]

  • -l 卷组上允许创建的最大逻辑卷数
  • -p 卷组中允许添加的最大物理卷数
  • -s 卷组上的物理的pe 大小
    • vgdisplay [卷组名] 显示改卷组的信息

VG扩容

vgextend [vg名称] [pv名称]

创建LVM 的逻辑卷

使用语法：lvcreate [参数] [逻辑卷名称]

  • -L 指定逻辑卷的大小，单位为（M G T）字节
  • -l 指定逻辑卷的大小（LE数）
    • lvscan 查看卷组信息
    • 逻辑卷创建完成后需要格式化与挂载才能使用

逻辑卷容量的变更

使用语法：lvresize [参数] 逻辑卷名称

  • -L 指定逻辑卷的大小，单位为（M G T）字节
  • -l 指定逻辑卷的大小（LE数）

逻辑卷扩容

流程:

  1. 查看是否有足够的vg空间：vgs
  2. VG扩容vgextend [vg名称] [pv名称]
  3. 使用命令扩容：lvextend -L +[内容大小] [lv的名称]
  4. 更新文件系统的大小：resize2fs [lv名称]

逻辑卷缩容

流程：

  1. 先确定缩小后的大小，并确保对应目标逻辑卷有足够的空间可容纳原有数据
  2. 卸载文件系统：umount
  3. 缩减文件系统：resize2fs [文件系统位置] [文件大小]
  4. 设置lv文件为不活动的：lvchange -a n [lv文件位置]
  5. 缩减逻辑卷：lvreduce
  6. 设置lv文件为活动的：lvchange -a y [lv文件位置]
  7. 检查文件系统：e2fsck -f [文件所在位置]
  8. 重新挂载使用

计划任务管理

at 绝对计时

  • hh.mm(小时，分钟)
  • midnight(深夜)
  • noon(中午)
  • teatime(饮茶时间，一般为下午4点)
  • 采用12小时制，在时间后面会加上AM 或者PM 来表明上午还是下午
  • 指定命令执行具体日期，month day(月日)mm/dd/yy(月/日/年)dd.mm.yy(日.月.年) 指定日期必须根在指定时间后面

相对计时 ：at 16:30 6/7/20

  • now+时间数量（如几天、几分钟等等） 时间 （min 分钟/hours 小时/ days 天/ weeks 星期）

使用语法：at [-V] [-q 队列] [-f 文件名] [-mldbv] 时间

查看作业详细

at -c [作业id]

删除作业

atrm 作业id

周期性任务

使用语法:crontab [参数]

  • -u 设置某个用户的cron服务
  • -l 列出某个用户cron服务的详细内容
  • -r 删除某个用户的cron
  • -e 编辑某个用户的cron服务

crontab 文件

crontab 文件需要输6个参数

minute hour day-of-month month-of-year day-of-week commands

参数 描述
minute 分钟（0-59）
hour 小时（0-23）
day-of-month 一个月的第几天（0-31）
month-of-year 一年的第几月（1-12）
day-of-week 一周的星期几（0-6）0周日
commands 执行命令
* 表示所有
/ 表示每如：/2表示为每两个单位
- 从某个数字到某个数字
， 散离数字

网络管理

网卡配置

重启网卡服务：systemctl restart NetworkManager

  • 当前设备状态：UP
  • link信息：设备的MAC地址
  • inet信息：IPV4地址子网掩码
  • 广播地址、作用域、设备名称
  • inet6信息：IPV6信息

查看网络性能

  • PX 表示接受的数据包信息
  • TX 表示传送的数据包信息

显示套接字信息

使用ss命令查看套接词统计信息，加上参数可过滤不同类型的套接字

  • h, –help 帮助
  • -V, –version 显示版本号
  • -t, –tcp 显示 TCP 协议的 sockets
  • -u, –udp 显示 UDP 协议的 sockets
  • -x, –unix 显示 unix domain sockets，与 -f 选项相同
  • -n, –numeric 不解析服务的名称，如 “22” 端口不会显示成 “ssh”
  • -l, –listening 只显示处于监听状态的端口
  • -p, –processes 显示监听端口的进程(Ubuntu 上需要 sudo)
  • -a, –all 对 TCP 协议来说，既包含监听的端口，也包含建立的连接
  • -r, –resolve 把 IP 解释为域名，把端口号解释为协议名称

ps -l

  • uid: 用户id
  • pid: 进程id
  • ppid: 父进程id
  • PRI 程序优先级，越小越优先
  • NI: 表示进程可被执行的优先级的修正数值
  • 优先级PRI范围

调整进程优先级

nice 使用语法

优先序adjustment 取值范围-19 - 20

nice [-n adjustent] [command [arg…]]

  • 把vi的运行优先级设置为-18

    nice -n -18 vi

renice 使用语法

优先序与nice 相同，作用对象有程序群组-g、进程-p、用户-u

renice [-n adjustment] [-] <pid>

  • 把vi 运行的优先级设置为10

    renice -n 10 -p 9700

前后台进程

  • 常用进程管理命令

    &： 将&放在命令的最后，可以把这个命令放到后台执行

    CTRL+z：当前台在执行一个命令的时候使用，会把它放到后台并暂停

    CTRL+c：中断当前在执行的命令

    jobs：查看当前有多少正在后台运行的命令

    fg：将后台的命令调到前台继续运行

    bg：将后台任务唤醒，并在后台运行

查看进程

查看进程 - ps、top

ps 命令可以显示当前进程，并列出详细的进程信息，字段含义如下表常用 ps aux 或 ps -ef 查看进程信息
ps 命令用于显示特定时间点的进程信息，top 命令可以实时显示进程信息

字段 描述
UID 用户识别符
PID 唯一进程识别符
%CPU 进程的 CPU 占用率
%MEM 进程的内存占用率
IN 进程谦让度，nice值
PRI 进程优先级
TTY 与进程关联的终端
STAT 进程的状态

top查看进程

  • top 常用参数

    -p 指定特定的pid进程号进行观察

列名 含义
PID 进程id
USER 程所有者的用户名
PR 优先级
nice 负值表示高优先级，正值表示低优先级
VIRT 进程使用的虚拟内存总量，单位kb。VIRT=SWAP+RES
RES 进程使用的、未被换出的物理内存大小，单位kb。RES=CODE+DATA
SHR 共享内存大小，单位kb
S 进程状态，D=不可中断的睡眠状态 R=运行 S=睡眠 T=跟踪/停止 Z=僵尸进程
%CPU 上次更新到现在的CPU时间占用百分比
%MEM 进程使用的物理内存百分比
TIME+ 进程使用的CPU时间总计，单位1/100秒
COMMAND 命令名/命令行
